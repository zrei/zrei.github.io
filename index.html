Game Projects

These are the game projects I've worked on. They are all developed in Unity, the game engine I am most familiar with. But I'm always willing to learn something new!

Star Rats

  • itch.io

Star Rats was a game developed for a 9-hour game jam hosted by my university's game development group. The theme of the game jam involved a bingo card of features that had to be integrated into the final game.

Working closely with my teammates, we designed and completed a game that was controlled with only two buttons. The player would control a sphere of rats and navigate through space, consuming cheese to grow in size while avoiding asteroids and cats. The goal of the game was for the player to reach 1,000,000 rats.

Responsible for:

  • Game states
  • Hooking up game logic with UI
  • Scaling the size and mass of the player object with number of rats
  • Tracking number of rats and providing an interface for interacting with the number of rats
  • Asteroid collision logic

This was the largest team I had worked in for a game project thus far, and it was incredibly fun to bounce ideas off of each other. Because of the short time-frame and the number of ideas we had, more compromises needed to be made, making communication between teammates vital.

Lost in the Sauce

  • itch.io
  • Github

Lost in the Sauce was a game developed for a week-long game jam hosted by my university's game development group. The theme of the game jam was "Magic as a Science".

Working closely with my teamates, we designed and completed an action game where the player navigates levels using basic attacks and skills, obtaining and upgrading skills in-between levels.

Responsible for:

  • Player movement
  • Player stats and how the game interacts with it
  • Activating skills, skills cooldown, skills interaction with enemies
  • Enemy movement
  • Projectile movement

Since this game jam lasted for a week, there was more time to generate numerous ideas that were more ambitious. However, I was taught the importance of developing a solid plan to pace ourselves while remaining flexible in order to adapt to unexpected situations. In the end, not all of our ideas could make it in, and we had to decide which were not feasible.

Arthur's Rage

  • itch.io
  • Github

Arthur's Rage was a game developed for a 9-hour game jam hosted by my university's game development group. This was a beginner game jam where all teams had to use the same assets but were otherwise free to develop any game they wanted.

Working closely with a small team, we developed a simple mob survival game with a twist. The player's sword would get larger with each felled enemy, and the player had to avoid slaying the Queen characters while continuing to slay the enemies.

Responsible for:

  • Player movement
  • Interactions between game objects
  • Sword expansion

This was my first game project after attending the introductory Unity lessons hosted by the game development group. It was an eye-opening experience that allowed me to put into practice the basic Unity skills I had picked up and further hone them.
