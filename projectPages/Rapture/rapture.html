Rapture

Rapture is a turn-based RPG with a focus on grid-based positioning and mechanics, developed in Unity with a team. It started as a project for a university module, but I and the rest of the team decided to continue working on it even after the module ended. It has been submitted to GDWC under the student project category.

I was responsible for the battle system:

  • Turn-based system, including calculation and tracking of turn order
  • Enemy AI (utilises a decision tree implemented in C#)
  • Player control - allowing the player to select skills, tiles, etc.
  • Tracking and representing unit data in battle, including status effects and skill cooldowns
  • Implementing the effects of various skills and their interactions with units
  • Data representation for skills, status effects, battles etc. that can be edited by designers

I was also responsible for the implementation of the world map:

  • Handling of level nodes based on progress
  • Implementation of cutscenes in the world map
  • Level unlock animation

Some editor tools I created:

  • A tool to check the validity of data e.g. looking for repeated IDs and replacing them with unique ones
  • A tool to easily navigate between the different scenes and easily load the game from the menu
  • A tool to help setup battle data

Other things I was responsible for:

  • Save management
  • Implementation of tutorial system
  • Some UI screens such as the party select screen

This was an extremely data-heavy game project. Extra consideration had to be given to the ease of accessing and editing data by the designer, which became a design challenge when considering the structures used to store the data. The cleanest design did not necessarily lend itself to easy data entry.

I was also completing my six-month UI game programmer internship while working on this game while it was still a graded school project, so it was a lesson in time management.
